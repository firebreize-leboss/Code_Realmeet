# CONSIGNES D'IMPLEMENTATION
# ============================================================

PHASE 1 — ANALYSE (ne code rien encore)

- Lis les 5 diagrammes pour comprendre l'architecture actuelle
- Analyse le point 3 (feature similaire) : retrouve les fichiers concernés, étudie le pattern utilisé (nommage, conventions, structure). Si "Aucune", cherche toi-même dans le code la feature la plus proche de ce qui est demandé
- Détermine toi-même à partir de l'ERD s'il faut une nouvelle table, de nouvelles colonnes, ou si l'existant suffit
- Détermine toi-même à partir de l'Infrastructure où placer le code (app, SQL/RPC, Edge Function, check-in server)
- Résume ton plan en 5 lignes max avant de commencer

PHASE 2 — IMPLEMENTATION

Implémente en respectant ces règles du projet :

- Reproduis le pattern de la feature similaire identifiée en phase 1
- SECURITY DEFINER : si tu crées une RPC qui bypass le RLS, filtre manuellement les blocked_users et les éléments cachés
- Soft delete : ne supprime jamais de données, utilise des colonnes status ou deleted_at
- Dual tables : slot_participants ET slot_group_members doivent toujours être cohérents quand la feature touche les participants
- Timezone : les dates d'activité sont stockées en heure Paris sans timezone info, le serveur est en UTC — convertis explicitement avec AT TIME ZONE 'Europe/Paris'
- UUID : utilise expo-crypto randomUUID(), jamais crypto.randomUUID()
- Pour chaque fichier modifié, vérifie que tout compile avant de passer au suivant

PHASE 3 — MISE A JOUR DES DIAGRAMMES

Après l'implémentation, mets à jour les diagrammes Mermaid impactés :

- domain-flow.mermaid : ajoute la feature dans le bon domaine (passe son status de planned à done, ou ajoute-la si elle n'existait pas). Ajoute de nouvelles connexions entre domaines si nécessaire
- erd.mermaid : ajoute les nouvelles tables/colonnes et leurs relations si tu en as créé
- infrastructure.mermaid : mets à jour si tu as ajouté une Edge Function, un nouveau service, ou une nouvelle dépendance externe
- navigation.mermaid : ajoute les nouveaux écrans/routes si tu en as créé
- user-flows.mermaid : ajoute ou modifie le flow si la feature introduit un nouveau parcours utilisateur, ou modifie significativement un parcours existant

Ne touche pas aux diagrammes qui ne sont pas impactés.

Dans app/(tabs)/activity.tsx, deux corrections : ÉTAPE 1 — Dans la fonction loadUserActivities, ajouter un filtre sur le status des participations. La requête `supabase.from('slot_participants').select('activity_id, slot_id').eq('user_id', currentUser.id)` doit ajouter `.in('status', ['active', 'completed'])` pour exclure les participations cancelled. Cela empêchera les activités dont l'utilisateur s'est désinscrit d'apparaître dans la liste. ÉTAPE 2 — Dans la même fonction loadUserActivities, lors du mapping des activités avec les slots, récupérer aussi is_cancelled depuis activity_slots. Modifier le select de `.select('date, time')` en `.select('date, time, is_cancelled')`. Ajouter un champ isCancelled dans l'objet retourné : `isCancelled: slotData?.is_cancelled || false`. Ajouter aussi isCancelled dans l'interface Activity en haut du fichier : `isCancelled?: boolean`. ÉTAPE 3 — Dans renderUserActivityItem, remplacer le badge statique "Terminé" par un badge conditionnel. Actuellement le code affiche toujours `<IconSymbol name="checkmark.circle.fill" ...><Text>Terminé</Text>` quand isPast est true. Modifier pour : si item.isCancelled est true, afficher un badge avec IconSymbol name "xmark.circle.fill" size 12 color "#EF4444" et texte "Annulé" avec color "#EF4444". Sinon garder le badge actuel "Terminé" avec les styles existants (checkmark.circle.fill, colors.textTertiary). Ne pas toucher au reste du fichier.

# Fix — Bug de redirection après vérification OTP téléphone

## Problème

Quand l'utilisateur vérifie son numéro de téléphone à l'étape 3 de l'inscription, l'app le redirige vers l'écran de sélection de compte (`/auth/account-type`).

### Cause racine

Dans `services/phone-verification.service.ts`, la méthode `verifyOtp()` fait :
1. `supabase.auth.verifyOtp({ phone, token, type: 'sms' })` → succès → Supabase crée une session temporaire
2. `supabase.auth.signOut()` → pour nettoyer cette session indésirable

Le problème : ce `signOut()` déclenche l'event `SIGNED_OUT` dans le listener `onAuthStateChange` de `contexts/AuthContext.tsx`. Ce listener met `user = null` et `profile = null`, ce qui provoque la redirection vers `/auth/account-type` via le guard d'authentification dans `app/(tabs)/_layout.tsx` et `app/index.tsx`.

### Logs confirmant le bug
```
[AUTH_DEBUG] onAuthStateChange event: SIGNED_OUT hasSession: false userId: undefined
[AUTH_DEBUG] No session -> clearing user & profile
[TAB SYNC] pathname: "/auth/account-type"
```

## Solution

Modifier `services/phone-verification.service.ts` pour ne PAS utiliser `signOut()`. À la place, utiliser une approche qui ne déclenche pas le listener d'auth.

### Approche : Flag de bypass dans le service

1. Dans `phone-verification.service.ts` : ajouter un flag statique `isVerifyingPhone` que le `AuthContext` peut consulter pour ignorer les events de session pendant la vérification OTP.

2. Dans `contexts/AuthContext.tsx` : dans le listener `onAuthStateChange`, vérifier ce flag. Si `isVerifyingPhone === true`, ignorer l'event `SIGNED_OUT` (ne pas mettre user/profile à null).

### Implémentation

**Fichier `services/phone-verification.service.ts`** — modifier la classe :

```typescript
class PhoneVerificationService {
  // Flag public pour que AuthContext puisse ignorer le signOut pendant la vérification
  public isVerifyingPhone = false;

  async sendOtp(fullPhoneNumber: string): Promise<{ success: boolean; error?: string }> {
    // ... inchangé
  }

  async verifyOtp(fullPhoneNumber: string, otpCode: string): Promise<{ success: boolean; error?: string }> {
    try {
      this.isVerifyingPhone = true; // ← Activer le flag AVANT verifyOtp

      const { data, error } = await supabase.auth.verifyOtp({
        phone: fullPhoneNumber,
        token: otpCode,
        type: 'sms',
      });

      if (error) {
        this.isVerifyingPhone = false; // ← Désactiver en cas d'erreur
        // ... gestion d'erreur existante (expired, invalid, etc.)
        return { success: false, error: '...' };
      }

      // Nettoyage : si une session a été créée, la détruire
      if (data?.session) {
        await supabase.auth.signOut();
      }

      // Petit délai pour laisser le temps au listener d'ignorer l'event
      await new Promise(resolve => setTimeout(resolve, 100));
      this.isVerifyingPhone = false; // ← Désactiver après le délai

      return { success: true };
    } catch (error: any) {
      this.isVerifyingPhone = false;
      return { success: false, error: error.message || 'Erreur réseau' };
    }
  }
}
```

**Fichier `contexts/AuthContext.tsx`** — modifier le listener :

```typescript
import { phoneVerificationService } from '@/services/phone-verification.service';

// Dans le useEffect avec onAuthStateChange :
const { data: { subscription } } = authService.onAuthStateChange(
  async (event, session) => {
    console.log('[AUTH_DEBUG] onAuthStateChange event:', event, 'hasSession:', !!session);

    // IMPORTANT : Ignorer le SIGNED_OUT si on est en train de vérifier un téléphone
    // (le signOut est déclenché par le nettoyage de la session OTP temporaire)
    if (event === 'SIGNED_OUT' && phoneVerificationService.isVerifyingPhone) {
      console.log('[AUTH_DEBUG] Ignoring SIGNED_OUT during phone verification');
      return;
    }

    // Ignorer aussi le SIGNED_IN si c'est la session OTP temporaire
    if (event === 'SIGNED_IN' && phoneVerificationService.isVerifyingPhone) {
      console.log('[AUTH_DEBUG] Ignoring SIGNED_IN during phone verification');
      return;
    }

    if (session?.user) {
      setUser(session.user);
      await loadProfile(session.user.id);
      if (event === 'SIGNED_IN') {
        notificationService.registerForPushNotifications();
      }
    } else {
      console.log('[AUTH_DEBUG] No session -> clearing user & profile');
      setUser(null);
      setProfile(null);
    }
    setLoading(false);
  }
);
```

## Fichiers à modifier

1. `services/phone-verification.service.ts` — ajouter le flag `isVerifyingPhone`, l'activer/désactiver dans `verifyOtp()`
2. `contexts/AuthContext.tsx` — importer `phoneVerificationService`, checker le flag dans le listener `onAuthStateChange` pour ignorer les events `SIGNED_OUT` et `SIGNED_IN` pendant la vérification

Implémente la logique complète de suppression d'ami avec blocage de conversation privée. Voici le comportement attendu : quand A retire B de ses amis, l'amitié est supprimée dans les deux sens, la conversation privée reste visible en lecture seule (historique conservé), et plus aucun des deux ne peut envoyer de message dans cette conversation tant qu'ils ne redeviennent pas amis. Voici les étapes précises :

ÉTAPE 1 — Fonction RPC remove_bidirectional_friendship : Crée dans Supabase (via SQL dans le dashboard ou migration) une fonction SECURITY DEFINER qui : supprime les deux lignes de friendships (user_id=caller/friend_id=target ET user_id=target/friend_id=caller), supprime la friend_request associée (dans les deux sens sender/receiver), et met à jour la conversation privée entre les deux utilisateurs en settant is_closed=true. Pour trouver la conversation privée, cherche dans conversations WHERE is_group=false et qui a exactement 2 participants (les deux users concernés) via conversation_participants. Le SQL exact :

CREATE OR REPLACE FUNCTION remove_bidirectional_friendship(p_friend_id uuid)
RETURNS void
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = 'public'
AS $$
DECLARE
  v_user_id uuid := auth.uid();
  v_conversation_id uuid;
BEGIN
  -- 1) Supprimer les deux sens de l'amitié
  DELETE FROM friendships 
  WHERE (user_id = v_user_id AND friend_id = p_friend_id)
     OR (user_id = p_friend_id AND friend_id = v_user_id);
  
  -- 2) Supprimer la friend_request associée
  DELETE FROM friend_requests
  WHERE (sender_id = v_user_id AND receiver_id = p_friend_id)
     OR (sender_id = p_friend_id AND receiver_id = v_user_id);

  -- 3) Trouver et fermer la conversation privée entre les deux
  SELECT cp1.conversation_id INTO v_conversation_id
  FROM conversation_participants cp1
  JOIN conversation_participants cp2 ON cp1.conversation_id = cp2.conversation_id
  JOIN conversations c ON c.id = cp1.conversation_id
  WHERE cp1.user_id = v_user_id 
    AND cp2.user_id = p_friend_id
    AND c.is_group = false;

  IF v_conversation_id IS NOT NULL THEN
    UPDATE conversations 
    SET is_closed = true 
    WHERE id = v_conversation_id;
  END IF;
END;
$$;

Dans app/activity-detail.tsx, modifier l'affichage pour les activités annulées : ÉTAPE 1 — Dans la fonction loadActivity, là où on récupère le slotInfo pour les activités passées (le bloc `if (passedSlotId)`), ajouter is_cancelled dans le select : changer `.select('date, time')` en `.select('date, time, is_cancelled')`. Créer un nouvel état avec useState : `const [isSlotSystemCancelled, setIsSlotSystemCancelled] = useState(false);`. Après avoir récupéré slotInfo, ajouter : `setIsSlotSystemCancelled(slotInfo?.is_cancelled || false);`. ÉTAPE 2 — Modifier le banner "Activité terminée". Actuellement le code est : `{isActivityPast && (<View style={styles.pastActivityBanner}><IconSymbol name="checkmark.circle.fill" size={20} color="#FFFFFF" /><Text style={styles.pastActivityBannerText}>Activité terminée</Text></View>)}`. Remplacer par un conditionnel : si isSlotSystemCancelled est true, afficher le banner avec backgroundColor '#EF4444' (rouge), IconSymbol name "xmark.circle.fill" size 20 color "#FFFFFF", et texte "Activité annulée". Sinon garder le banner actuel "Activité terminée" avec le style existant. ÉTAPE 3 — Masquer la section "Votre avis" quand le slot est annulé. La section review est conditionnée par `{isActivityPast && !isHost && !isBusiness && (...)}`. Ajouter la condition `&& !isSlotSystemCancelled` pour que ça devienne `{isActivityPast && !isHost && !isBusiness && !isSlotSystemCancelled && (...)}`. Ainsi on ne peut pas noter une activité qui a été annulée par le système. Ne pas toucher au reste du fichier.

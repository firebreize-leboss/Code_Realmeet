## TÂCHE — components/button.tsx : remplacer l'opacity press par un scale press

### Contexte du bug
Sur Android, les variantes `outline` et `ghost` ont un `backgroundColor: "transparent"`. 
L'effet press actuel utilise `opacity: pressed ? 0.8 : 1` ce qui rend le contenu 
semi-transparent sur le fond blanc → le bouton "disparaît" visuellement.

On veut un effet de zoom subtil (scale down) au press, comme un vrai bouton physique.

### Ce qu'il faut faire

1. Lis le fichier `components/button.tsx` en entier avant toute modification.

2. Ajoute les imports Reanimated nécessaires :
   - `Animated, useSharedValue, useAnimatedStyle, withSpring` depuis `react-native-reanimated`
   - `motion` depuis `@/styles/motionTokens`

3. Remplace le `<Pressable>` par un `<Animated.View>` wrappant un `<Pressable>` :
   - Crée un `useSharedValue(1)` pour le scale
   - Crée un `useAnimatedStyle` qui retourne `{ transform: [{ scale: withSpring(scaleValue.value, motion.spring.snappy) }] }`
   - Le `<Animated.View>` reçoit le style animé
   - Le `<Pressable>` utilise `onPressIn` pour setter `scale.value = motion.scale.press` (0.97)
   - Le `<Pressable>` utilise `onPressOut` pour setter `scale.value = motion.scale.normal` (1)

4. IMPORTANT — Supprime le `opacity: pressed ? 0.8 : 1` de la ligne style du Pressable.
   Garde uniquement `opacity: disabled ? 0.6 : 1` (l'opacity ne sert plus que pour le disabled).

5. Garde `android_ripple` tel quel.

6. Le `style` prop externe doit rester sur le Pressable (pas sur l'Animated.View wrapper).

### Pattern de référence (déjà utilisé dans le projet)

Regarde comment c'est fait dans `components/ActivityCard.tsx` et `components/FloatingTabBar.tsx` :
ils utilisent exactement ce pattern Animated.View + Pressable + onPressIn/onPressOut + motion tokens.

### Structure résultante attendue
```tsx
const scale = useSharedValue(1);
const animatedStyle = useAnimatedStyle(() => ({
  transform: [{ scale: withSpring(scale.value, motion.spring.snappy) }],
}));

return (
  <Animated.View style={animatedStyle}>
    <Pressable
      onPress={onPress}
      onPressIn={() => { scale.value = motion.scale.press; }}
      onPressOut={() => { scale.value = motion.scale.normal; }}
      disabled={disabled || loading}
      style={[
        getVariantStyle(),
        {
          height: sizeStyles[size].height,
          paddingHorizontal: sizeStyles[size].padding,
          borderRadius: sizeStyles[size].borderRadius,
          opacity: disabled ? 0.6 : 1,
        },
        style,
      ]}
      android_ripple={{ color: colors.primary + '30', borderless: false }}
    >
      {/* contenu inchangé */}
    </Pressable>
  </Animated.View>
);
```

### Ce qu'il ne faut PAS faire
- Ne PAS garder `opacity: pressed ? 0.8 : 1` — c'est la source du bug
- Ne PAS ajouter de Haptics
- Ne PAS toucher au reste du fichier (styles, variants, getTextColor, etc.)
- Ne PAS utiliser le `({ pressed })` callback de style du Pressable pour autre chose que le style statique
